# Язык L_tri_call

Как `L_tri_if`, но с надстройкой вызовов в выражениях `let`.

## Формат модуля

Программа — список функций:

```lisp
(
  (func main
    (bb0 ...)
    (bb1 ...)
  )
  (func helper ...)
)
```

Интерпретатор исполняет только функцию `main`.

## Инструкции в BB

- `(let vN <expr>)`
- `(goto bbX)`
- `(if-goto vC bbT else bbF)`
- `(return <expr>)`

Phi-узлы разрешены в начале блока:

```lisp
(let v3 (phi (bb1 v1) (bb2 v2)))
```

## Вызовы

Поддерживаются только built-in вызовы арифметики/сравнений в форме:

```lisp
(call + a b)
(call - a b)
(call * a b)
(call / a b)
(call < a b)
(call > a b)
(call = a b)
```

`a` и `b` могут быть переменными или константами. Другие цели `call` сейчас считаются ошибкой.

## `nil`

`nil` допускается как специальный литерал и интерпретируется как `0`.
Это нужно для случаев, когда SSA дампит `SSA_VAL_VOID`.
