# Язык L_tri_call

Как `L_tri_if`, но с надстройкой вызовов в выражениях `let`.

## Формат модуля

Программа — список функций:

```lisp
(
  (func main
    (bb0 ...)
    (bb1 ...)
  )
  (func helper v0 v1
    (bb0 ...)
  )
)
```

Аргументы функции перечисляются прямо в заголовке `func` между именем функции и первым базовым блоком.

Интерпретатор исполняет только функцию `main`, поэтому `main` должна быть без аргументов.

## Инструкции в BB

- `(let vN <expr>)`
- `(goto bbX)`
- `(if-goto vC bbT else bbF)`
- `(return <expr>)`

Phi-узлы разрешены в начале блока:

```lisp
(let v3 (phi (bb1 v1) (bb2 v2)))
```

## Вызовы

Поддерживаются built-in вызовы арифметики/сравнений в форме:

```lisp
(call + a b)
(call - a b)
(call * a b)
(call / a b)
(call < a b)
(call > a b)
(call = a b)
```

`a` и `b` могут быть переменными или константами.

Также поддерживаются вызовы пользовательских функций:

```lisp
(call helper x y)
```

Аргументы вычисляются в окружении вызывающей функции, затем создаётся новый фрейм локальных переменных, в котором параметры целевой функции получают вычисленные значения.

## `nil`

`nil` допускается как специальный литерал и интерпретируется как `0`.
Это нужно для случаев, когда SSA дампит `SSA_VAL_VOID`.
